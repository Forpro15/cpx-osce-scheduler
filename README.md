# CPX/OSCE 실기 연습 자동배정 (Constraint-based Scheduler)

의과대학 국가고시 실기(CPX/OSCE/하이브리드) 연습 일정을 **제약 조건 기반으로 자동 배정**하기 위한 규칙 명세와 설정 템플릿입니다. 한 학교의 내부 운영 노하우를 다른 의대 학생회가 그대로 포크해서 재사용할 수 있도록 일반화했습니다.

> **상태(Status): 동작하는 참조 구현 포함.** (1) 제약 모델 명세, (2) 학교별 설정 템플릿, (3) 설정만 넣으면 배정 결과를 산출하고 하드 제약을 자동 검산하는 **파이썬 솔버**(`scheduler.py`)를 제공합니다. 개인정보를 제거한 스프레드시트 참조 템플릿은 로드맵에 있습니다.

이 저장소와 여기서 파생되는 어떤 파일에도 **학생 개인정보를 넣지 마세요.**

- 실제 학생 명단이 담긴 워크북은 로컬에만 두고, `.gitignore`가 실수 커밋을 막습니다

## 이게 무엇인가 / 무엇이 아닌가

**이것입니다**
- CPX/OSCE/하이브리드 연습 배정의 **하드 제약 + 소프트 목표**를 명시적으로 정리한 규칙 세트
- 학교마다 다른 값(조 구성, 방 개수, 제외일, 시간 구조)을 채우면 되는 **설정 템플릿**
- "왜 이렇게 배정되는가"를 검증 가능하게 만든 **검산 항목** 목록

**이것은 아닙니다**
- 완성된 원클릭 솔버가 아닙니다 (참조 구현은 스프레드시트 수식 기반)
- 학생 데이터를 담는 저장소가 아닙니다
- 특정 학교 전용이 아닙니다 (아래 예시는 경희대 26년도 값이며, 그대로 바꿔 쓰면 됩니다)

## 제약 모델 한눈에

| 구분 | 예시 (경희대 26년도) |
|------|------|
| 배정 기간 | 2026-07-06 ~ 08-05 |
| 하루 구조 | 4개 Class (08:00~22:00) |
| 운영 | 평일 Class 3·4 / 주말 Class 2·3·4 |
| 시설 | 방 15개, Class당 ≤ 30명, 방 = ⌊인원/2⌋ |
| 조 구성 | 16개 조 / 84명 |
| 하루 제한 | 한 조 = 하루 최대 1 Class |
| OSCE | 한 Class에 한 조만, 임상수기센터 배타 사용 |
| 균등 목표 | 전 조 연습 횟수 균등 (원본 결과: 12회 · 편차 0) |

전체 규칙과 근거는 [`docs/constraints.md`](docs/constraints.md)를 보세요.

## 우리 학교에 맞게 쓰는 법 (Adaptation)

1. `config/schedule.example.yaml`을 복사해 `config/schedule.yaml`로 만듭니다.
2. 우리 학교 값으로 수정합니다:
   - `period` — 연습 기간
   - `classes` / `operating_classes` — 시간 구조와 요일별 운영 Class
   - `capacity` — 방 개수, Class당 최대 인원
   - `groups` — 조 개수와 조별 인원수 *(이름 없이 숫자만)*
   - `cpx_blackout_dates` — 시험·행사로 배정 불가한 날짜
   - `osce` — OSCE 연습 기간과 조별 배치표
3. [`docs/constraints.md`](docs/constraints.md)의 검산 항목으로 결과를 검증합니다.

---

## 실행 (Reference solver)

```bash
pip install pyyaml
python scheduler.py config/schedule.yaml            # 배정 + 검산 결과 출력
python scheduler.py config/schedule.yaml --csv out/schedule.csv
```

솔버는 조 번호와 인원수만 사용하며 학생 개인정보를 다루지 않습니다. 예시 설정으로 실행하면 전 조 균등(편차 0) 배정과 하드 제약 전 항목 PASS를 확인할 수 있습니다.
