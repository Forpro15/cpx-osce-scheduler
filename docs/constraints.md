# 제약 모델 명세 (Constraint Model)

CPX/OSCE/하이브리드 연습 배정 문제를 **하드 제약(반드시 충족)**과 **소프트 목표(가능한 한 최적화)**로 나눠 정의합니다. 모든 값은 `config/schedule.yaml`에서 학교별로 바꿉니다. 괄호 안은 경희대 26년도 예시값입니다.

---

## 1. 대상과 기간
- 배정 대상: CPX, OSCE, 하이브리드
- 배정 기간: `period.start` ~ `period.end` (2026-07-06 ~ 08-05)
- 종료일 이후 데이터는 마스터에서 숨김/삭제

## 2. 하루 시간 구조
하루는 여러 개의 **Class**(시간 블록)로 구성됩니다. (예: 4개 Class, 08:00~22:00)

| Class | 시간(예) |
|-------|---------|
| 1 | 08:00 ~ 11:30 |
| 2 | 11:30 ~ 15:00 |
| 3 | 15:00 ~ 18:30 |
| 4 | 18:30 ~ 22:00 |

## 3. 하드 제약 (Hard Constraints)

### H1. 운영 시간
- 평일: `operating_classes.weekday` 만 운영 (예: 3·4)
- 주말: `operating_classes.weekend` 만 운영 (예: 2·3·4)
- `cpx_excluded_classes`는 CPX 배정에 사용하지 않음 (예: Class 1 — 단 OSCE 예외 존재)

### H2. 시설 수용
- 방 총개수 ≤ `capacity.rooms_total` (예: 15)
- 한 Class 총 배정 인원 ≤ `capacity.max_people_per_class` (예: 30)
- 방 개수 = ⌊인원 ÷ `capacity.people_per_room`⌋ (예: 인원/2)

### H3. CPX 제외일
`cpx_blackout_dates`에 속한 날은 CPX를 전혀 배정하지 않음. 각 날짜에 사유를 함께 기록합니다(시험 전일/당일, OSCE, 임종평가 등).

### H4. 하루 1 Class
- 한 조는 하루 최대 1개 Class에만 배정
- 같은 Class에 같은 조 중복 배정 금지

### H5. OSCE 규칙
- OSCE 연습은 `osce.practice_window` 안에서만 배정
- **한 Class에 한 조만** (`one_group_per_class`)
- 연습 기간에는 `osce.placement` 표에 따라 조를 고정 배치
- OSCE 연습이 잡힌 조는 **그날 CPX 배정 제외** (H4와 연동)

### H6. 임상수기센터 배타성
- OSCE와 하이브리드는 모두 임상수기센터를 사용
- 따라서 **같은 Class에 OSCE 조와 하이브리드 조가 공존 불가**
- 결과적으로 임상수기센터는 Class당 최대 1개 조

## 4. 소프트 목표 (Soft Goals)
가능한 한 만족시키되, 하드 제약과 충돌하면 하드가 우선합니다.

- **S1. 균등 횟수**: 모든 조의 총 CPX 연습 횟수를 균등하게 (원본 결과: 전 조 12회, 편차 0)
- **S2. 시간대 분산**: 각 조가 운영 Class를 고르게 경험 (특정 시간대 편중 방지)
- **S3. 기간 분산**: 앞쪽 몰림·빈 날 없이 기간 전체에 고르게 배분
  - 일별 쿼터를 그날 운영 Class 수(평일 2 / 주말 3)에 비례해 분배
- **S4. 연속 반복 완화**: 같은 조가 동일 시간대에 연속 반복 배정되지 않도록

## 5. 검산 항목 (Verification)
배정 결과가 유효한지 자동으로 검증할 목록입니다. 하나라도 실패하면 배정이 무효입니다.

- 조별 CPX 연습 횟수 편차 = 0 (S1)
- 모든 Class 인원 ≤ `max_people_per_class` (H2)
- 방 개수 ≤ `rooms_total` (H2)
- OSCE Class당 조 수 ≤ 1 (H5)
- 모든 제외일의 CPX 인원 합 = 0 (H3)

## 참고: 참조 구현 형태
원본은 스프레드시트로 구현되어 있습니다. 마스터 시트에서 파생 시트(종합교육관/임상수기센터/조별 캘린더)를 **수식으로 미러링**하고, 별도 **검산 시트**가 위 항목을 함수로 자동 검증합니다(마스터 수정 시 즉시 갱신). 개인정보를 제거한 참조 워크북 공개는 로드맵 참고.
